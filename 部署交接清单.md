# 酷太产品自动回复部署交接

## 当前服务

- 钉钉机器人：Node.js 20 容器，Compose 项目名 `kutai-auto-reply`，健康端口 `3000`。
- LLM Wiki：同一 Compose 中的 `llm-wiki-watcher` 监听 Obsidian 原始知识；保存稳定后自动编译并运行基础检索测试。
- 回复链路：LLM Wiki 只提供产品事实证据，DeepSeek 只基于证据组织语言；没有可靠证据即进入审核，不自由补全。

## 目录

- `KT-automatic-reply-repo`：机器人源码、Docker Compose、测试和运行文档。
- `产品知识库`：Obsidian 事实源、客服规则和 `LLM-Wiki` 索引。
- `图片库`：产品原图、安装资料及图片索引。
- `.env`、`data/`、`runtime/`：本机机密和运行状态，不提交 GitHub。

## 首次检查

1. 在 `.env` 填写钉钉 Stream、DeepSeek 和视觉模型参数。
2. 运行 `docker compose up -d --build`。
3. 确认 `http://localhost:3000/` 健康。
4. 在控制台确认 LLM Wiki 显示“索引最近生效时间”和页数。
5. 先在 `PENDING_REVIEW_MODE=true` 下完成回归测试，再决定是否开启自动发送。
