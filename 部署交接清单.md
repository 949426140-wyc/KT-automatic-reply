# 酷太产品自动回复部署交接

## 当前服务

- Dify：1.14.2，Docker Compose 本地部署，Web 端口 `8088`。
- 钉钉机器人：Node.js 20 容器，Compose 项目名 `kutai-auto-reply`，健康端口 `3000`。
- 机器人默认 `PENDING_REVIEW_MODE=true`，在 Dify 知识库、DeepSeek 和视觉模型未验收前不自动发送。

## 目录

- `dify/docker`：Dify 官方 1.14.2 部署文件和持久化数据。
- `钉钉自动回复`：机器人源码、Dockerfile、Compose、运行配置、审核队列与审计日志。
- `Dify知识库导入包`：待上传的精准知识卡与规则。
- `产品知识库源`：机器人本地兜底用产品矩阵和图文 JSON。
- `图片库`：产品图、安装资料和 PDF 素材。

## 待用户完成

1. 打开 Dify，创建管理员账户。
2. 创建“酷太知识库”并上传 `Dify知识库导入包/05_精准知识卡/_Dify上传合集`。
3. 创建 Chatflow/聊天助手应用，绑定知识库，生成应用 API Key。
4. 将 Dify API Key、DeepSeek API 和视觉模型参数写入 `钉钉自动回复/.env`。
5. 先运行待审核模式完成回归测试，再决定是否开启自动发送。
