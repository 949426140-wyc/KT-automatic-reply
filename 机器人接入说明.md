# 酷太钉钉机器人接入说明

## 当前状态

- 本地机器人服务文件：`server.js`
- 启动脚本：`start-bot-reply.bat`
- 停止脚本：`stop-bot-reply.bat`
- 回调路径：`/dingtalk/webhook`
- 本地端口：`3000`
- 个人号轮询：已通过 `auto-reply.disabled` 禁用
- 审计日志：`bot-reply-audit.jsonl`

## 钉钉侧需要创建的内容

创建钉钉机器人时，选择 Stream 模式。

Stream 模式不需要公网回调地址，也不需要内网穿透。本地服务会用 Client ID 和 Client Secret 与钉钉建立长连接。

## .env 需要补的字段

在开发者后台的「凭证与基础信息」页面复制：

```env
DINGTALK_APP_KEY=
DINGTALK_APP_SECRET=
DINGTALK_AGENT_ID=
DINGTALK_ROBOT_CODE=
```

字段对应关系：

- `DINGTALK_APP_KEY`：页面里的 Client ID（原 AppKey / SuiteKey）
- `DINGTALK_APP_SECRET`：页面里的 Client Secret（原 AppSecret / SuiteSecret）
- `DINGTALK_AGENT_ID`：页面里的原企业内部应用 AgentId，后续主动发应用消息可能用到
- `DINGTALK_ROBOT_CODE`：机器人消息里通常会带；如果页面能看到机器人编码，也可以填上作为兜底

机器人策略已经配置：

```env
DINGTALK_CONNECT_MODE=stream
BOT_GROUP_ONLY_AT=true
BOT_MAX_MESSAGE_AGE_MIN=5
BOT_AUDIT_LOG_FILE=D:\酷太\钉钉自动回复\bot-reply-audit.jsonl
```

含义：

- 群聊只处理 @ 机器人的消息。
- 只处理 5 分钟内的新消息，避免历史消息被补回。
- 每条机器人消息和处理结果都会写审计日志。

## 启动与停止

启动：

```powershell
D:\酷太\钉钉自动回复\start-bot-reply.bat
```

停止：

```powershell
D:\酷太\钉钉自动回复\stop-bot-reply.bat
```

健康检查仍然保留本地 HTTP 端口：

```text
http://localhost:3000/
```

## 处理链路

钉钉机器人消息进入 `server.js` 后，会复用现有自动回复引擎：

1. 消息时间与 @ 过滤
2. 员工、订单、物流、售后、上架、价格等规则过滤
3. 图片/上文图片识别
4. LLM Wiki 知识检索与 DeepSeek 基于证据的回复生成
5. 本地规则兜底，例如安装教程、产品图片需求
6. 机器人发送回复
7. 写入 `bot-reply-audit.jsonl` 审计日志
