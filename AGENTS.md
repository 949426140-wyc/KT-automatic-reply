# 产品自动回复项目约束

本项目根目录：`D:\AI\产品自动回复`

## 自动回复分层职责

- 本地程序负责运行机制与安全过滤：消息类型判断、订单/物流/售后等问题类型过滤、员工/群聊过滤、AI 生成后的关键词质量过滤、去重、日志、平台连接和发送。
- 本地过滤词、允许词和生成后质量判断直接维护在 `钉钉自动回复\auto-reply.js`；这类规则可以修改本地代码。
- Dify 知识库 Markdown 只维护产品相关内容：产品事实、规格、尺寸、材质、安装、选型、兼容关系、产品图片归属和具体产品问题的标准回答口径。
- 不把“建议 / 通常 / 一般”等通用允许词或禁止词清单写入 Dify 产品知识库。
- 不把订单、物流、售后、人员等通用问题类型过滤规则混入 Dify 产品知识卡。

## 当前维护位置

- 本地自动回复与过滤：`钉钉自动回复\auto-reply.js`
- 运行说明：`钉钉自动回复\钉钉自动回复规则.md`
- Dify 产品知识上传合集：`Dify知识库导入包\05_精准知识卡\_Dify上传合集`
- 产品知识源及具体产品口径：`Dify知识库导入包` 下的产品 Markdown/精准知识卡

## 修改判断

- 调整“哪些类型的问题不进入 AI、哪些生成词需要拦截”：改本地 JS。
- 调整“某个产品是什么、尺寸是多少、能否安装、如何选型、正确产品答案是什么”：改 Dify 产品知识库 MD。
- 修改知识库后同步更新 `_Dify上传合集`，由用户重新上传 Dify。
- 修改本地 JS 后重建并验证钉钉机器人容器。
