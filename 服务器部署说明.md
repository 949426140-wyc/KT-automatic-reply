# 酷太钉钉自动回复服务

## 链路

钉钉 Stream 消息 → 本地硬过滤/去重/上下文 → 可选视觉识别 → LLM Wiki 产品证据检索 → DeepSeek 基于证据生成候选回复 → 待审核或发送 → 审计日志。

## 首次配置

1. 复制 `.env.example` 为 `.env`。
2. 填写钉钉 Stream 的 `DINGTALK_APP_KEY` / `DINGTALK_APP_SECRET`。
3. 填写 `AI_API_KEY`（DeepSeek）以及视觉模型参数。
4. 确认同级 `产品知识库/LLM-Wiki/index/knowledge-index.json` 已生成。
5. 首次测试保持 `PENDING_REVIEW_MODE=true`，确认审核队列无误后再改为自动发送。

## 启动

```bash
docker compose up -d --build
docker compose logs -f kutai-dingtalk-bot
```

健康检查：`http://localhost:3000/`。

## 服务器迁移

整个目录可复制到 Linux/Windows 服务器。服务器只需 Docker Compose；`.env` 和 `data/` 是需单独备份的配置与持久化数据。
